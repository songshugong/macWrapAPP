# FinderWrapNavigator（PoC）

一个纯 Swift 的 Finder 键盘导航补偿工具，不依赖 Hammerspoon/Karabiner。

目标行为：

- 在 Finder 图标视图里按 `→` 时，先执行默认右移。
- 如果选中项没有变化（通常是当前行已到最右侧），自动执行：
  - 直接一次性选中“下一行最左”
  - 如果已经是整页最后一个图标，则截断（保持不动）
- 在 Finder 图标视图里按 `←` 时，先执行默认左移。
- 如果选中项没有变化（通常是当前行已到最左侧），自动执行：
  - 直接一次性选中“上一行最右”
  - 如果已经是整页第一个图标，则截断（保持不动）
- 结果上接近 Windows 的“行尾右移自动换到下一行”。

## 运行

```bash
cd /Users/songzihan/Documents/New\ project

```

启动后会出现菜单栏项 `FinderWrap`。

## 权限

首次运行需要系统权限，否则无法监听/发送按键：

- 系统设置 -> 隐私与安全性 -> 辅助功能
- 系统设置 -> 隐私与安全性 -> 输入监控（推荐）

## 当前实现边界

- 仅在 Finder 成为前台时启用监听，切到其他应用会关闭事件 tap。
- 仅处理 Finder 前台时、无修饰键的 `←/→`。
- 按住 `Option` 时会临时禁用增强逻辑，立即回到 Finder 原生行为。
- 按住 `←/→` 连续触发时会做重复事件合并，降低高频重算导致的抖动与漏判。
- 长按 `→` 会在约 `0.22s` 后进入加速档，提升跨行跳转检查频率（仅增强模式生效）。
- 菜单提供“长按→加速模式”开关，可手动开启/关闭该加速行为。
- 加速档下在满足边界条件时可触发一次“跟进跳转”，让连续长按的纵向移动更快。
- 文件重命名/文本编辑状态下会自动绕过接管，保持 Finder 原生文本光标行为。
- 边界跳转增加“二次确认+一次轻量重试”，降低偶发选中失败。
- 仅在 Finder 普通窗口生效，桌面场景（Desktop）不接管。
- 仅在“图标视图特征”命中时接管（容器角色排除 `AXOutline/AXBrowser/AXTable`，并校验网格行距与单元尺寸）。
- 不规则网格自适应列匹配目前默认关闭（保持严格边界语义：右到下一行首、左到上一行尾）；可在源码开关中手动启用。
- 行内普通移动仍走 Finder 原生逻辑，只有边界换行/回滚由本工具接管。
- 默认关闭 fallback 多步按键模拟，优先降低视觉闪烁（失败时回退 Finder 原生行为）。
- 性能优化：网格缓存 TTL 动态调整（约 120ms~280ms），连续按键时减少 AX 遍历开销。
- 单次处理采用预算控制（约 14ms），超预算时回退到 Finder 原生行为，避免输入卡住。
- 内置轻量诊断计数（命中率、放行率、失败率、缓存命中率等）。
- 内置资源计数（CPU/内存采样，平均 CPU、峰值 CPU、当前/峰值内存）。
- 菜单新增 `打印统计信息`，可查看并输出运行报告。
- 基于辅助功能树读取“当前选中项”，Finder 大版本 UI 变化后可能需要微调。
- 这是 PoC 版，尚未做自动启动和签名分发。

## 路径级策略配置

在源码中编辑以下两个数组（`FinderRightArrowWrapService.swift`）：

- `wrapEnabledPathPrefixes`
- `nativeOnlyPathPrefixes`

规则：

- `nativeOnlyPathPrefixes` 命中时，强制使用 Finder 原生导航。
- 如果 `wrapEnabledPathPrefixes` 为空，则默认全路径启用（除非被 native-only 命中）。
- 如果 `wrapEnabledPathPrefixes` 非空，则仅命中这些前缀的目录启用 wrap。
